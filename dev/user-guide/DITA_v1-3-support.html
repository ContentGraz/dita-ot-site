DITA 1.3 support

DITA Open Toolkit 2.2 provides preliminary processing support for the upcoming OASIS DITA 1.3 specification. Initial preview support for this specification was added in version 2.0 of the toolkit; version 2.2 extends this foundation to support key scopes and branch filtering along with additional DITA 1.3 features.

Because DITA 1.3 is fully backwards compatible with previous DITA DTDs and schemas, DITA-OT 2.2 provides the proposed 1.3 materials as the default DTDs for processing. The XML Catalog resolution maps any references for DITA 1.2 doctypes to the 1.3 DTDs, for example. All processing ordinarily dependent on the 1.0, 1.1, or 1.2 definitions continues to work as usual, and any documents that make use of the newer DITA 1.3 elements or attributes will be supported with specific new processing.

Because this support is based on a yet-to-be-approved version of the proposed OASIS DITA 1.3 specification, if you choose to investigate any 1.3-based function, be aware that the 1.3 implementation in this version of the toolkit is preliminary. Upon final approval of the DITA 1.3 standard, toolkit developers will review the implementation to make certain that it conforms to the final specification.

Initial Preview Support for DITA 1.3 in DITA-OT 2.0

The following DITA 1.3 features were implemented in version 2.0 of the toolkit. Issue numbers correspond to the tracking number in the GitHub issues tracker.

  • Support DITA 1.3 link syntax (milestone 2) #1649
  • Support DITA 1.3 cascade attribute (milestone 2) #1636
  • Implement DITA 1.3 profiling (milestone 2) #1635
  • Add new DITA 1.3 highlighting elements (milestone 4) #1651
  • Add DITA 1.3 markup and xml domain support (milestone 4) #1652
  • Add DITA 1.3 div element (milestone 4) #1654

Additional DITA 1.3 support in DITA-OT 2.2

The following DITA 1.3 features were implemented in version 2.2 of the toolkit.

  • Initial support for DITA 1.3 key scopes #1979, #1648
  • Initial implementation of DITA 1.3 branch filtering #1969, #1637
    The implementation is a separate module that is run before keyref processing. The process
    • Splits branches so that each branch contains a single ditavalref
    • Generates @copy-to attributes for each branch-generated <topicref>
    • Filters the map based on branch filters
    • Rewrites duplicate generated copy-to targets with a numbered -# suffix
    • Copies and filters generated copy-to targets
    • Filters topics that were not branch-generated
  • The proposed DITA 1.3 grammars are now used as the default DTDs for processing #2094
  • The DOTJ065 warning that appeared when branch filter–generated topics were renamed has been changed to an “info” message to clarify that renaming is intentional. The new DITA 1.3 elements <dvrResourcePrefix> and <dvrResourceSuffix> can be used to control renaming and avoid this message if necessary. #2096
  • Processing order has been revised to process same topic fragments before the conref phase, to enable content references to elements in the same topic using a reference such as <p conref="#./ID"/> as reported in #1649. #1968
  • Keyref processing has been extended to support multiple scope names in a single @keyscope attribute #2004
Note: For the latest status information on DITA 1.3-related features, see the DITA 1.3 label in the GitHub issues tracker.
